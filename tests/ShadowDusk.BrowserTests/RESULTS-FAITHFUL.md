The served `shadowdusk-dxc` `[JSImport]` module is the **FAITHFUL pinned DXC→WASM** shim (`src/ShadowDusk.Wasm/wwwroot/shadowdusk-dxc.js` + `dxc/dxcompiler.{js,wasm}`), NOT the Slang sample shim. The Faithful (mode-2) section below is the Gate G2 deliverable: each shader is compiled **entirely in-browser** (`WasmShaderCompiler.CompileAsync`: faithful DXC→WASM → SPIRV-Cross WASM → managed reflect/write → `.mgfx`), then loaded via `new Effect(gd, bytes)` in KNI WebGL and pixel-compared against the desktop DesktopGL render of ShadowDusk's own bytes (`references-sd/`). Because the faithful WASM SPIR-V is byte-identical to desktop DXC (gates G0/G1), the in-browser `.mgfx` equals the desktop `.mgfx`, so the renders must match within tolerance.
