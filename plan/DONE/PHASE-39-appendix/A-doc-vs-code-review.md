### 2.5 `src/ShadowDusk.HLSL/FxPreParser.cs` — the FULL API
- Entry: `public static Result<FxParseResult, FxParseError> Parse(string source, string sourceFile)` (:150–157). Private ctor; lexes with `FxLexer`, computes char offsets, runs `ParseFile()`.
- `FxParseResult` (Ast/FxParseResult.cs): `{ string StrippedHlsl; IReadOnlyList<TechniqueInfo> Techniques; IReadOnlyList<SamplerInfo> Samplers; IReadOnlyList<ParameterAnnotation> ParameterAnnotations }`.
- `TechniqueInfo` (Ast/TechniqueInfo.cs): `{ Name, Span, IReadOnlyList<PassInfo> Passes, IReadOnlyList<AnnotationEntry> Annotations, bool IsEffect11 }`.
- `PassInfo` (Ast/PassInfo.cs): `{ Name, Span, string? VertexEntryPoint, string? PixelEntryPoint, string? VertexProfile, string? PixelProfile, IReadOnlyList<RenderStateEntry> RenderStates, IReadOnlyList<AnnotationEntry> Annotations }`.
- `SamplerInfo` (Ast/SamplerInfo.cs): `{ Name, SamplerType, string? TextureReference, IReadOnlyList<SamplerStateEntry> StateEntries, Span }`. `SamplerStateEntry`/`RenderStateEntry` are `record(string Key, string Value, SourceSpan Span)`.

**(a) Profile capture: YES** — `ParsePass` (:719–755) stores the lowercased raw token. **Caveat (load-bearing):** parse happens pre-preprocess (pipeline Stage 1 :54 vs Stage 2 :87), so for the Phase 17 corpus the stored profile is the macro NAME `"ps_shadermodel"`, not `ps_3_0`. The FNA backend must own profile selection (clamp vs_3_0/ps_3_0, or default ps_2_0; optionally honor literal SM1–3 profiles validated by `IsKnownProfile` :43, currently unused).

**(b) Rewrites + clean skippability: YES.** All rewrites are independent `if`-branches in the `ParseFile` main loop (:290–545); technique stripping (:297–331), parameter-annotation stripping (:453–496) and sampler-metadata capture are separate concerns. A mode flag (e.g. `Parse(source, sourceFile, FxSourceMode mode)` with `Modern` default — additive overload, zero churn for existing callers) gates:
- :344–391 sampler_state Form 1: ALWAYS still run `ParseSamplerDecl` (metadata). In FNA mode, replace the whole decl with the original `samplerNd S;`-style decl (initializer stripped) **for both used and unused samplers** — current code ERASES unused Form-1 decls (:384–387) and rewrites used ones to SamplerState+Texture2D; both behaviors are wrong for FNA. Alternative: pure passthrough if probing shows vkd3d-1.18's HLSL frontend accepts `= sampler_state {…}` initializers under the d3dbc target — verify first; initializer-stripping is the safe default.
- :393–414 bare-sampler rewrite: skip (passthrough — `sampler s0;` / `: register(s0)` is native D3D9 HLSL).
- :432–447 legacy `texture T;` rewrite: skip; verify vkd3d accepts the `texture` object type under d3dbc, else erase the decl (binding survives in metadata).
- :503–517 COLOR→SV_Target: skip.
- :527–540 tex2D→`.Sample`: skip (and the `_samplerTextureBindings` population becomes metadata-only).
Technique/pass stripping and annotation stripping stay ON in FNA mode (vkd3d-as-plain-HLSL-compiler must not see fx constructs; the metadata is already captured). Line-number preservation machinery (`BuildStrippedOutput` :1002–1071, `BuildDeclReplacement` :1230–1241) is reused as-is.

**(c) sampler_state contents recorded: YES** — `SamplerInfo.TextureReference` (the `<T>` form, parsed :857–884) + `StateEntries` (raw MinFilter/MagFilter/AddressU… key/values, :885–895). This is exactly the fx_2_0 sampler-parameter payload. `CompilationPipeline` currently never reads `fxParsed.Samplers` — the FNA branch is its first consumer (sampler→texture binding for object-table shader-sampler mapping; state entries for the sampler's state assignments).

### 2.9 `src/ShadowDusk.Core/MgfxWriter.cs` — patterns to mirror in `Fx2EffectWriter`
Place `Fx2EffectWriter` in `ShadowDusk.Core` beside `MgfxWriter` (Core has no project deps; writer stays pure/unit-testable). Mirror: (1) `public Result<byte[], ShaderError> Write(<IR>, <options record>)` — validation up-front returning coded `ShaderError`s (SD00xx) instead of throwing (:23–43); (2) options as a small positional record (`MgfxWriterOptions(MgfxProfile Profile, byte MgfxVersion = 10)`); (3) **body-first serialization** into a `MemoryStream` then header derived from body (:45–65) — fx_2_0 analog: the offset-based object table needs the same two-phase build (write parameter/technique tables with placeholder offsets, or build object list first and back-patch); (4) `BinaryWriter` over `MemoryStream` with UTF-8 gives little-endian primitives — correct for fx_2_0 too, but **fx_2_0 strings are length-prefixed (u32) ASCII + NUL, 4-byte aligned — do NOT reuse `BinaryWriter.Write(string)`** (7-bit-length .NET format used by MGFX); write a dedicated helper; (5) deterministic output (constraint #3) — `ManagedMd5` available if any hash is wanted, though fx_2_0 has none; (6) static private write-helpers per table, sentinel/count conventions documented inline with MojoShader (`mojoshader_effects.c`) as the cited spec, the way MgfxWriter cites MonoGame's EffectReader.
